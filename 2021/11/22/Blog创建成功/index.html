Blog创建成功

发表于 2021-11-22 更新于 2023-2-9 分类于 随笔
本文字数： 894 阅读时长 ≈ 1 分钟

两天的尝试，我终于通过 github+hexo 成功制作出了我自己的blog：sunjunge08.github.io，这篇文章也是 blog 的开篇之作。下面，我要分几个部分来叙述一遍我建立 blog 的全过程。

之前建立的一些 blog

早在去年，我就已经有了亿些用来写骚话文章的地方。最开始是用 write.as（注：现在已挂），后来我觉得它的功能太少，就放弃了，同时也把所有文章都删掉了。

后来，我在 B 站、知乎、贴吧等地方也写了一些专栏。但那不是我发布《骚话》（这本“书”我在后面还会提到）的好地方，而且不太安全（你懂的），遂放弃。但是里面的文章我保留了下来，有时间可以传到这个 blog 中。

再后来，时间到了2021年8月。那时我看到一篇教程，是用 gitblog 创建个人博客，具体可以到 sjg-blog 了解。然而，它也有很多缺点。第一，它使用 github 的 actions 功能，可以在写完 issues 之后自动把信息写入 README.md 中，但后来就不能用了，很多次都是失败的。

第二，github 经常被墙，所以有时候上不去，这就很蛋疼了。即使科学上网，也有一定的风险。于是，我也把它放弃了。同样地，里面的文章我会慢慢搬运过去的。

为什么要建立这个 blog

原因很简单，就是为了替代前面所说的各种“blog”。

最近我在翻知乎的时候，偶然在我的收藏夹里找到了这样一篇文章：《技术小白搭建个人博客 github+hexo》。于是，我心血来潮，在2021年11月21日（星期日）下午，用了不知道多长时间，按照文章中的做法建立blog。但是失败了，没有达到我的预期。我一气之下把 github 的 repository 删掉了，包括我本地的珍贵文件。

那天晚上，学校发来通知，第二天（也就是我现在写文章的这一天）雪休。于是，我重新建立了 blog ，终于成功创建了新的 blog。

这个 blog 会写什么

目前的打算有这样几个：

（1）搬运之前在 gitblog 上的文章；
（2）改进网站的 UI ，使之更美观；
（3）继续写骚话。

最后，欢迎大家访问我的博客。
