Internet主要下载方式及下载器介绍

发表于 2022-5-15 更新于 2023-2-9 分类于 网络
本文字数： 8.8k 阅读时长 ≈ 8 分钟

导语

这篇文章将从各个方面入手，讲解常见的 Internet 下载方式以及对应的下载器。此外还会介绍一些特殊信息的下载方式。

1. 下载方式

在互联网上，我们经常需要下载一些信息和文件。为了下载到它们，我们首先要获取它们的下载地址。其中，最常见的就是 HTTPS 地址。此外，还有各种其它的文件分享方式，例如 BitTorrent 和 Magnet 等。

1.1 HTTP 和 HTTPS

1.1.1 HTTP

超文本传输协议（英语：HyperText Transfer Protocol，缩写：HTTP）是一种用于分布式、协作式和超媒体信息系统的应用层协议。HTTP是万维网的数据通信的基础。1

设计 HTTP 最初的目的是为了提供一种发布和接收 HTML 页面的方法。通过 HTTP 或者 HTTPS 协议请求的资源由统一资源标识符（Uniform Resource Identifiers，URI）来标识。

HTTP 的发展是由蒂姆·伯纳斯-李于1989年在欧洲核子研究组织（CERN）所发起。HTTP 的标准制定由万维网协会（World Wide Web Consortium，W3C）和互联网工程任务组（Internet Engineering Task Force，IETF）进行协调，最终发布了一系列的 RFC，其中最著名的是1999年6月公布的 RFC 2616，定义了 HTTP 协议中现今广泛使用的一个版本—— HTTP 1.1。

2014年12月，互联网工程任务组（IETF）的 Hypertext Transfer Protocol Bis（httpbis）工作小组将 HTTP/2 标准提议递交至 IESG 进行讨论，于2015年2月17日被批准。HTTP/2 标准于2015年5月以 RFC 7540 正式发表，取代 HTTP 1.1 成为 HTTP 的实现标准。

举例：http://example.com/

1.1.2 HTTPS

超文本传输安全协议（英语：HyperText Transfer Protocol Secure，缩写：HTTPS；常称为 HTTP over TLS 、HTTP over SSL 或 HTTP Secure）是一种通过计算机网络进行安全通信的传输协议。HTTPS 经由 HTTP 进行通信，但利用 SSL/TLS 来加密数据包。HTTPS 开发的主要目的，是提供对网站服务器的身份认证，保护交换资料的隐私与完整性。这个协议由网景公司（Netscape）在1994年首次提出，随后扩展到互联网上。2

历史上，HTTPS 连接经常用于万维网上的交易支付和企业信息系统中敏感信息的传输。在2000年代末至2010年代初，HTTPS 开始广泛使用，以确保各类型的网页真实，保护账户和保持用户通信，身份和网络浏览的私密性。

另外，还有一种安全超文本传输协议（S-HTTP）的 HTTP 安全传输实现，但是 HTTPS 的广泛应用而成为事实上的 HTTP 安全传输实现，S-HTTP 并没有得到广泛支持。

举例：https://sunjunge08.github.io/

1.2 BitTorrent

BitTorrent 协议（简称 BT，俗称比特洪流、BT 下载）是用在对等网络中文件分享的网络协议程序。和点对点（point-to-point）的协议程序不同，它是用户群对用户群（peer-to-peer），而且用户越多，下载同一文件的人越多，下载该文件的速度越快。且下载后，继续维持上传的状态，就可以“分享”，成为其用户端节点下载的种子文件（.torrent），同时上传及下载。3

举例：Life Stinks.torrent

1.3 Magnet

磁力链接（Magnet URI scheme），是对等网络中进行信息检索和下载文档的电脑程序。和基于“位置”连接的统一资源定位符不同，磁力链接是基于元数据（metadata）文件内容，属于统一资源名称。也就是说，磁力链接不基于文档的 IP 地址或定位符，而是在分布式数据库中，通过散列函数值来识别、搜索来下载文档。因为不依赖一个处于启动状态的主机来下载文档，所以特别适用没有中心服务器的对等网络。尽管它符合开源标准，却被垄断信息资源和监控用户行为的个人、组织、企业和政府拒之门外。4

举例：magnet:?xt=urn:btih:77786833D9B722F41C886449DD8BEA626D321E41

2. 下载器

对于上述几种下载方式，我们需要使用不同的下载器来下载它们。下面是几个典型的例子。

2.1 IDM / FDM / NDM

这三种下载器都是 HTTP(S) 下载器，FDM 有下载 BT 资源的功能，但是并不是很推荐。

2.1.1 IDM

2.1.1.1 介绍

Internet Download Manager（简称 IDM）是一个用于 Windows 系统的下载管理器。5

IDM 可以让用户自动下载某些类型的文件，可将文件划分为多个下载点以更快下载，并行出最近的下载，方便访问文件。相对于其他下载管理器，具有独特的动态文件分割技术。其优点在于，能分析下载地址中的实际地址，但若地址无效时则跳换成失效网页，当地址重新有效而无法恢复成原来的地址。而缺点在于，必须设置分段的临时文件存放的磁盘位置，下载完成后再自动合并成一个完整的文件，同时删除临时文件（这种做法不同于其他下载软件预先分配一整块磁盘空间）。所以，下载的如果是机密文件，就容易被数据恢复软件恢复，造成安全隐患。IDM可被调用于 Internet Explorer、Safari、Google Chrome、Opera、Netscape Navigator 以及 Mozilla Firefox（火狐）浏览器中。

相对于其它下载管理工具，它具有独特的动态档案分区技术。它的出色之处在于，它能分析下载地址中的实际地址，但若地址无效时则跳换成失效网页，当地址重新有效可自动下载完成。聪明的 in-speed 技术会动态地将所有设定应用到某种联机类型，以充分利用下载速度。6

官方网站：https://www.internetdownloadmanager.cn/

2.1.1.2 使用方法

在 IDM 的官方网站上，你可以看到：IDM 是一个收费软件，不过有30天的免费试用期。

这里提供一个破解版，是由俄罗斯大神 elchupacabra 在知名软件破解论坛 Lrepacks 上发布的。这个版本目前没有什么问题，能够正常使用，这里是：下载地址。7

以前，由于 IDM 官方打击破解和盗版，对大部分的 IDM 破解版本都做出了弹窗提示，迫使破解用户换正版。这一举动引起了很大的讨论，当时有两个方法解决这一问题：一是换成较低版本的 IDM 破解版，没有弹窗；二是用弹窗拦截和浏览器脚本来关掉它。但这两种方式都挺麻烦的。后来这个弹窗终于被俄罗斯大神给去掉了，也就是上面的 v6.40.11 版。

安装完毕后，在 View->Language 中设置语言为简体中文。如果 IDM 没有自动安装浏览器集成（初次下载一般都会安装，但是后续更新有时就不会），可以在其安装目录中找到相应的 .crx 文件进行安装。

2.1.1.3 常用功能

IDM 之所以强大，不仅是因为其下载速度快，还因为它有很多实用功能。这是很多下载器和浏览器所没有的。这里来介绍一下。

在 IDM 的“帮助”选项中，可以看到有“帮助内容”“教程””计划任务及下载队列”“站点抓取帮助”这四个选项。你可以点击它们，然后在 IDM 官网上下载到 IDM 的帮助文件，并从中学习 IDM 的更多使用方法。

此外，还可以在 Internet Download Manager 中文博客、使用技巧、常见问题、新闻资讯 这几个页面中了解更多关于 IDM 的使用方法和技巧。

2.1.2 FDM

2.1.2.1 介绍

Free Download Manager（简称 FDM）是一款跨平台的下载管理器。8

FDM 最初是专有软件，但随着 2.5 版发布成为自由软件。从 3.0.850 版（2010年4月15日）开始，源代码不再打包提供下载，但可以从项目的 Subversion 资源库获取，从 3.9.x 及以后版本不再提供源代码（不再开源，5.0 版本以后移除 GNU 自由软件协议）。Free Download Manager 荣获2007年 Webuser 金奖。

官方网站：https://www.freedownloadmanager.org/zh/

2.1.2.2 不足之处

在 IDM 开始弹窗警告的时候，我决定弃用 IDM 转为 FDM，因而 FDM 成为了我那时下载文件的主力。但是，后来我发现了它的一个很大的问题：无法自定义 User-Agent。这也就意味着，我无法下载百度网盘的文件了（见 3.2.3）。这可不行！后来我实在忍不下去了，就换成了 NDM，可没想到 NDM 也有很多问题，具体我会在 2.1.3.2 部分详细说明。

2.1.3 NDM

2.1.3.1 介绍

Neat Download Manager（简称 NDM）是一款轻量级免费下载工具，也是 macOS 和 Windows 的免费下载管理器和加速器，可使用动态分段算法下载文件。它支持 HTTP，HTTPS 和 FTP 协议，HTTP 代理和 SOCKS 协议，HTTP 和代理认证，暂停/继续功能以及过期下载的自动续订。它具有浏览器扩展，用于捕获下载链接和从许多网站下载视频/音频。免费下载整洁的下载管理器。9

Neat Download Manager 是一个可靠的应用程序，与其他类似的下载应用程序相比，它可以为您提供快速安全的下载速度。它能够与 Safari，Google 和 Opera 等不同的搜索引擎进行协作，并为您提供管理和保存声音和电影格式的不同文件的功能。即使您正在下载文件，它也使您可以为下载设置带宽限制。该程序支持拖放功能，并且与所有类型的流行网络浏览器兼容，包括 Google，Yahoo，Mozilla Firefox，Chrome，Opera 等。该程序还支持批处理，使您可以同时下载多个文件而不会降低速度。

官方网站：http://www.neatdownloadmanager.com/index.php/en/

2.1.3.2 不足之处

前面讲到，我弃用了 FDM 转为了 NDM，但是不久我就发现它有两个问题：

没有中文版本。这个下载器比较小众，甚至在 维基百科 / Wikipedia 上都没有对它的介绍。而且其网站做的也挺简陋，没有太多内容。经过很长时间的搜寻，我也没找到它的中文版。直到后来，我在一个个人网站 大眼仔旭 上找到了由此网站的站长汉化的软件。下载下来一看，质量非常高。于是，这第一个问题就解决掉了。

经常报错，无法下载某些链接。有一次，我用 B站 的一个油猴脚本 Bilibili-Evolved 下载视频时，发现 NDM 弹窗报错了。

就像这样，而且在下载 Pxxnhub 的视频时也出现了此类情况。这个问题我就百思不得其解了。于是我到 Bilibili-Evolved 的 Discussion 页面发起了一个讨论10，希望那里会有大神帮忙解决。然而，如你所见，并没有人知道报错的原因。直到现在，此问题仍尚未解决。

2.1.4 三种下载器的比较

已经有人将它们做了对比和测试，文章地址是：《同比 6 款热门下载器，IDM 还是最好用的那个吗？》。

这里直接给出我的结论：（1）IDM 最好用。无论是下载速度和稳定程度，FDM 和 NDM 都没法与其相比。（2）IDM 没有什么大问题，就是不定时地会受到官方的弹窗提示。不过这都是小问题，总会有破解大神解决的。（3）FDM 的速度很不稳定，忽高忽低，经常调回 1KB/s ，就很难受。（4）FDM 和 NDM 都有不能下载的情况，上面也说过了，所以不推荐这两个。

2.2 BitComet

2.2.1 介绍

比特彗星（BitComet，缩写：BC）是一个用 C++ 语言为 Microsoft Windows 平台编写的 BitTorrent 客户端软件，也可用于 HTTP/FTP 下载，并可选装“ eMule 插件”通过电驴网络进行 BitTorrent / 电驴 同时下载。11

官方给出的特点是这样的12：

BT 下载高速而且功能强大。
长效种子: 独有的长效种子功能，能显著增加下载速度 (0.93新功能)。
支持磁链: 使用 DHT 网络，无需 Torrent 文件即可进行 BT 下载。
边下载边播放: 支持视频预览，在下载 avi, rmvb, wmv 等视频文件过程中可以边下载边播放。
跨协议下载: BT 任务可以自动查找 HTTP / FTP / EMULE 下载源，提高下载速度。
文件分块对齐: 创新的文件分块对齐制作种子，兼容旧版而且不增加任何冗余数据传输。
智能连接优化: 自动根据您的网络连接优化下载。
智能磁盘缓存: 使用内存作下载缓存, 有效减小硬盘读写速度, 延长其使用寿命。

支持 DHT 网络: 通过公用 DHT 网络，无需连接 Tracker 也能进行 BT 下载。
自动配置防火墙，支持全自动配置 Windows 网络连接防火墙（ICF）和网络连接共享（ICS）。
自动配置路由器端口映射，采用 UPnP 技术实现内网免配置（要求路由器支持 UPnP）。
支持多 Tracker 协议，UTF-8 扩展，UDP Tracker v2 协议。

官方网站：http://www.bitcomet.com/cn

2.3 Aria2

2.3.1 Aria2

Aria2 是一款自由、跨平台命令行界面的下载管理器，该软件根据 GPLv2 许可证进行分发。支持的下载协议有：HTTP、HTTPS、FTP、BitTorrent 和 Metalink。14

官方网站：https://aria2.github.io/

2.3.2 AriaNG GUI

Aria2 本身是一个命令行工具，并没有图形化界面，因此想要使用就必须敲代码。但这实在是太麻烦了，对不懂程序的人很不友好。于是，有人制作了一个图形化工具，完美包含了 Aria2 的各种功能，叫做 AriaNG，它是一个让 Aria2 更容易使用的现代 Web 前端。AriaNg 使用纯 HTML & JavaScript 开发，所以其不需要任何编译器或运行环境。您只要将 AriaNg 放在您的 Web 服务器里并在浏览器中打开即可使用。AriaNg 使用响应式布局，支持各种计算机或移动设备。15

AriaNG 的发布地址为：https://github.com/mayswind/AriaNg/releases

但是，AriaNG 只能在浏览器中使用，比较不方便。因此，又有人制作了一个 Aria2 图形界面客户端，叫做 AriaNG GUI，它有如下特性16：

  • 集成 Aria2c（默认 Aria2 RPC：http://localhost:6800）
  • 多线程下载
  • 未完成任务退出自动保存
  • 支持 PT/BT
  • 下载完成消息通知
  • 多语言支持
  • 支持配置多个 Aria2 RPC
  • 支持打开下载文件夹, 在文件管理器中显示已下载的文件（仅限使用内置的 Aria2 RPC）
  • 使用响应式布局，支持各种计算机或移动设备
  • 友好的界面交互
    • 任务排序（按文件名，文件大小，进度，剩余时间，下载速度等），文件排序，连接节点排序
    • 任务搜索
    • 拖拽调整下载顺序
    • 更详细的任务信息（健康度，连接节点客户端信息等）
    • 根据文件类型过滤任务中的文件（按视频，音频，图片，文档，应用程序，存档文件等）
    • 全局以及单任务的上传/下载速度图表
    • 完整支持 Aria2 设置选项

AriaNG GUI 可以当做一个独立客户端来使用，不依赖浏览器，而且与 AriaNG 的功能完全一致，方便快捷。

AriaNG GUI 的发布地址为：https://github.com/mayswind/AriaNg/releases

3. 特殊下载

当然，只有上述下载方式是远远不够的，某些资源不是想要下载就可以下载的，有时还会受到版权、服务器带宽以及其他各种因素的影响。在这一部分，我将具体说明各种资源的获取方式。如果不做特殊说明，下面获得的下载地址都可以用上文提到的下载器进行下载。

3.1 影视资源

日常生活中，最常见的一类资源就是影视资源（包括但不限于电影、电视剧、动漫、纪录片以及音乐等），这类资源一般是“可以观看而无法下载”。下面是这类资源的下载方式。

目前，国内的视频网站主要有爱优腾和B站（国外的视频网站这里暂且不提，将会在3.3部分说明），它们都可以用同一种办法进行下载，那就是使用油猴脚本。

接下来，介绍一下 Tampermonkey。Tampermonkey 是最受欢迎的用户脚本管理器，用户数量超过一千万，支持 Chrome，Microsoft Edge，Safari，Opera Next 和 Firefox 等浏览器。17

3.1.1.3 Bilibili-Evolved

对于 B 站的视频，还有一种方法可以下载，那就是使用脚本 Bilibili-Evolved。Bilibili-Evolved 曾经也在 Greasy Fork 上发布并更新，但是后来突然无法安装。于是我到 Bilibili-Evolved 的 GitHub Issues 中询问18，得到的回复是：“自 v1.10.14 版后, 脚本的大小超出了 Greasy Fork 规定的最大脚本大小（2000000字符），因此 Greasy Fork 这边只能永远停在 v1.10.12 版了。”

所以，现在的安装地址应是：Bilibili Evolved #安装。可以选择正式版和预览版，我用的是正式版。

安装后打开视频《【春晚鬼畜】赵本山：我就是念诗之王！【改革春风吹满地】》，点开左边的功能栏→下载视频。

选择好参数后，可以选择输出方式。这里推荐三种：

（1）Toast：弹一条消息显示出下载链接，右键新标签页打开就可以下载。链接有 referer 限制，复制无用，且不能保留视频文件名。只能用浏览器自带的下载器下载。

（2）Aria2 RPC：使用 Aria2 RPC 功能发送下载请求。下面的参数保持默认即可，需要提前打开 AriaNG GUI（见2.3.2部分）。

（3）IDM：使用 IDM 的 .ef2 格式导出，可以在 IDM 中使用导入开始下载。导入方式为：进入 IDM →任务→导入→从“IDM 导出文件”导入。

注意：这种方法与自身账号有关，也就是说，无法下载大会员独享的番剧（账号自身是大会员则可以下载）。
